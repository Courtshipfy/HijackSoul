# 室内场景系统 — 程序开发任务清单

来源：《【室内场景系统】多视角切换与交互物功能》v0.1

---

## 1. 场景与视角系统
- 子场景结构与背景图管理
- 多场景之间的视角切换流程（初版：直切，预留多种切换方式扩展）
- 切换触发区（矩形碰撞盒，含目标场景、切换方式、方向提示、音效）
- 场景切换黑场过渡（0.2s 淡入/淡出，遮罩层级最高）

## 2. 玩家交互系统
- 鼠标控制的点击/悬停检测
- 可交互范围判定（以鼠标为中心的区域）
- 悬停高亮（半透明外轮廓）与名称提示（白字黑描边）
- 点击/悬停音效播放
- 切换瞬间禁用所有交互

## 3. 可交互物系统
- 交互物基类与四种类型实现：
  - 剧情触发类
  - UI 面板类
  - 状态切换类
  - 拾取类
- 通用属性：ID、名称、范围、消耗机制（一次性/无限次）、初始状态、音效
- 一次性交互物的"已交互"表现（灰暗/消失/换样貌）
- 重叠物件按层级优先检测，上层消耗后下层暴露

## 4. 交互触发与系统对接
- 与剧情系统对接（传 剧情ID）
- 与 UI 系统对接（传 面板ID + 上下文数据）
- 与场景状态机对接（传 状态机ID + 目标状态）
- 与背包/拾取系统对接（道具入包）

## 5. 互斥与异常处理
- 剧情播放中 / UI 开启中：屏蔽所有交互与场景切换
- 切换中：屏蔽所有交互物点击
- 鼠标在交互范围但被屏蔽时：可选播放轻微拒绝音效
- 快速来回穿越切换区的防抖

## 6. 数据与配置
- 场景配置表（场景ID、名称、背景、BGM、环境音）
- 切换区配置表（切换ID、所属场景、目标场景、切换方式、音效）
- 交互物配置表（ID、所属场景、名称、类型、触发参数、范围、位置、是否一次性、初始状态、音效）
- Excel → JSON 数据导出与加载

## 7. UI / UX 反馈
- 场景切换黑场遮罩
- 悬停高亮与名称 Tooltip
- 拾取提示条（右下角"获得【XX】"，1.5s 自动消失）
- 切换方向/目标场景描述提示
- 异步加载时的 Loading 图标

## 8. 资源加载
- 背景图常规加载
- 大图异步加载机制（黑场期间等待加载完成再淡出）

## 9. 状态持久化
- 一次性交互物的"已交互"标记
- 场景内状态机（开灯、开柜等）当前值

## 10. 调试与验收
- 跑通多场景间切换无异常
- 四类交互物分别能正确调用对应系统
- 一次性交互物消耗后不再响应
- 剧情/UI 中切换被正确拦截
- 重叠物件按层级正确暴露
