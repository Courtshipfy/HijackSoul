# 项目架构文档

**Godot 数据驱动游戏框架**

> 当前阶段：主题未公布 · 基础搭建中
> 
> 团队：策划 + 美术 + 音频 + 程序（2人）

---

## 核心原则

策划改 Excel → 工具转 JSON → 游戏读取 → 模块处理 → UI 显示

---

## 第 1 章 · 当前阶段说明

### ⚠️ 主题未公布 · 基础搭建阶段

当前 GameJam 主题尚未公布，所有工作集中在「基础设施」和「通用系统」搭建。主题公布后，只需填充 `modules/` 玩法逻辑和 `levels/` 关卡内容，框架不动。

---

### 📋 本阶段可搭建内容（主题无关）

| 模块 | 内容 |
|------|------|
| `autoload/` 全局服务 | event_bus · scene_manager · audio_manager · save_manager · data_manager · game_state |
| `data/` 数据层 | JSON 加载器框架 + 示例数据文件（给策划看格式） |
| `modules/` 空壳 | player 节点树（无移动逻辑）· inventory/quest/dialog 空目录 |
| `ui/` 通用界面 | main_menu · pause_menu · settings · common 控件 |
| `levels/` 测试关卡 | test_level 空白场景，用来测试移动和 UI |
| `glue/` 胶水框架 | game_bootstrap 启动顺序 · module_connector 空连接 |
| `tools/` 转换器 | `_Tools/converter.py` Excel→JSON 工具链 |
| `Pipeline/` 规范 | 命名规则 · 导出规则 · 数据字段规范 |

---

### ⛔ 本阶段不可做（等主题公布）

- ❌ 写具体玩法逻辑（敌人 AI、伤害公式、任务条件）
- ❌ 设计关卡内容（地图布局、敌人摆放、剧情流程）
- ❌ 填正式 Excel 表（没有主题，不知道需要什么数据）
- ❌ 做正式美术资源（不知道风格方向）

---

### 🔄 主题公布后的接入方式

1. 确定核心玩法循环（如：潜行→收集→逃脱）
2. 填充 EventBus 事件（`player_died` · `item_collected` · `quest_completed`）
3. 填充 `modules/` 逻辑（player 移动 · inventory 增删 · dialog 解析）
4. 填充 `glue/` 连接（捡到物品→发信号→背包加→UI 更新）
5. 填充 `levels/` 摆放（地图、敌人出生点、道具位置）
6. 策划填正式 Excel 表，转换器生成 JSON 进游戏

```

    │
    ▼

    │
    ▼

    ▼
```

```
    ▼

```

```

    │
    ▼

    │
    ▼

    │
    ▼
```

## 第 6 章 · 关键规则

### 六条铁律

| 编号 | 规则 | 违反后果 |
|------|------|---------|
| 1 | **策划只改 Excel，不手改 JSON** | 下次转表会覆盖，丢失修改 |
| 2 | **美术源文件放 `_Assets/`，最终文件放 `assets/`** | Godot 加载不了外部资源 |
| 3 | **模块之间不直接引用，只发信号** | 代码缠成一团，改一个地方崩五个地方 |
| 4 | **glue 只连接，不写业务逻辑** | 胶水层膨胀成第二个万能管理器 |
| 5 | **关卡只摆放，不写复杂逻辑** | 关卡脚本变成隐藏的系统代码 |
| 6 | **data/ 和 assets/ 只读，运行时写入 user://** | 版本冲突，数据丢失 |

### 典型场景：玩家捡到物品 → 背包更新 → UI 显示

```
┌─────────────┐    signal     ┌─────────────┐    signal     ┌─────────────┐
│  modules/    │ ─────────────→│  EventBus   │ ─────────────→│  modules/    │
│  player      │  item_collected│  (autoload) │  inventory_  │  inventory   │
│              │                │             │  updated      │              │
└─────────────┘                └─────────────┘                └─────────────┘
                                      │
                                      ▼
                               ┌─────────────┐
                               │  glue/       │
                               │  connector   │
                               │  （转发）     │
                               └─────────────┘
                                      │
                                      ▼
                               ┌─────────────┐
                               │  ui/         │
                               │  inventory   │
                               │  _panel      │
                               │  （更新显示） │
                               └─────────────┘
```

### 通信规则

**模块内部：**
- 可以直接引用自己的脚本和节点

**模块之间：**
- 不访问对方内部变量和节点
- 可以通过公开 API
- 可以通过 Godot signal
- 广播型事件用 EventBus

**glue：**
- 只连接模块
- 不判断玩法规则

**autoload：**
- 只提供基础服务
- 不承载具体玩法
