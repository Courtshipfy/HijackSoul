# 交互物与物品配置说明

本文档给策划使用，说明如何在 Godot 场景里新增和配置可点击物品。

## 一、基本做法

在房间场景的 `ObjectLayer` 下拖入：

```text
res://modules/interaction/interactive_object.tscn
```

拖入后，移动根节点 `InteractiveObject` 的位置即可摆放物品。

## 二、常用字段

选中 `InteractiveObject` 根节点，在 Inspector 里配置：

- `object_id`：唯一 ID，必须填写。例：`prototype.room_front.key`
- `display_name`：物品名称。例：`钥匙`
- `hover_text`：鼠标悬停提示。例：`拾取钥匙`
- `sprite_texture`：物品图片。
- `sprite_offset`：图片偏移。
- `sprite_scale`：图片缩放。
- `hotspot_size`：可点击范围大小。
- `hotspot_offset`：可点击范围偏移。

## 三、拾取物品

如果点击后要进入背包，在 `default_actions` 里配置：

```gdscript
[
  {
    "type": "pickup_item",
    "item_id": "prototype_key"
  },
  {
    "type": "show_toast",
    "message": "获得钥匙"
  }
]
```

效果：

- 物品加入右侧背包。
- 场景里的物品隐藏。
- 自动存档。

## 四、普通查看物

如果点击后只显示提示：

```gdscript
[
  {
    "type": "show_toast",
    "message": "这是一张旧照片。"
  }
]
```

## 五、触发 NarrRail 对白

如果点击后触发故事对白：

```gdscript
[
  {
    "type": "emit_story_event",
    "event_id": "inspect_wall_note"
  }
]
```

`event_id` 需要提前和程序确认，由 `StoryBridge` 映射到对应 NarrRail 剧情。

## 六、需要背包物品的交互

例如一个锁盒：

- 空手点击：提示锁住。
- 选中钥匙点击：消耗钥匙并打开。

`default_actions`：

```gdscript
[
  {
    "type": "show_toast",
    "message": "盒子锁住了。"
  }
]
```

`item_interactions`：

```gdscript
[
  {
    "item_id": "prototype_key",
    "actions": [
      {
        "type": "remove_item",
        "item_id": "prototype_key"
      },
      {
        "type": "set_flag",
        "flag_id": "box_opened",
        "value": true
      },
      {
        "type": "show_toast",
        "message": "盒子打开了。"
      }
    ]
  }
]
```

## 七、命名规则

`object_id` 建议格式：

```text
章节.房间.物品
```

例：

```text
chapter_01.room_01.key
chapter_01.room_01.locked_box
chapter_01.room_01.wall_note
```

注意：

- `object_id` 不能重复。
- 已经上线或已经用于存档的 `object_id` 不要随便改。
- 可拾取物品的 `item_id` 也要稳定。

## 八、检查清单

新增交互物后，请确认：

- [ ] 放在当前场景的 `ObjectLayer` 下。
- [ ] 填了唯一的 `object_id`。
- [ ] 填了 `display_name` 和 `hover_text`。
- [ ] 设置了 `hotspot_size`，鼠标容易点中。
- [ ] 配置了正确的 action。
- [ ] 运行后 hover 有提示。
- [ ] 点击后反馈符合预期。
