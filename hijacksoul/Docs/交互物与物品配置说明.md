# 交互物与物品配置说明

本文档给策划使用，说明如何在 Godot 场景里新增和配置可点击物品。

## 一、基本做法

优先在房间场景的 `ObjectLayer` 下拖入对应预制体：

```text
res://modules/interaction/prefabs/inspect_object.tscn
res://modules/interaction/prefabs/pickup_object.tscn
res://modules/interaction/prefabs/view_exit_object.tscn
res://modules/interaction/prefabs/story_trigger_object.tscn
res://modules/interaction/prefabs/puzzle_entry_object.tscn
res://modules/interaction/prefabs/locked_object.tscn
```

拖入后，移动根节点位置即可摆放物品。

如果不确定类型，仍然可以使用通用对象：

```text
res://modules/interaction/interactive_object.tscn
```

细分预制体会在 Inspector 里隐藏无关 action 字段，只保留当前类型需要配置的内容。通用对象会显示完整高级 action 配置。

## 二、常用字段

选中 `InteractiveObject` 根节点，在 Inspector 里配置：

- `object_id`：唯一 ID，必须填写。例：`prototype.room_front.key`
- `display_name`：物品名称。例：`钥匙`
- `hover_text`：鼠标悬停提示。例：`拾取钥匙`
- `sprite_texture`：物品图片。
- `sprite_offset`：图片偏移。
- `sprite_scale`：图片缩放。
- `hotspot_size`：可点击范围大小。
- `hotspot_offset`：可点击范围偏移。
- `object_kind`：对象类型，用于调试和校验。使用细分预制体时会自动设置。

## 三、推荐对象类型

| 类型 | 预制体 | 用途 |
|------|--------|------|
| 查看物 | `inspect_object.tscn` | 点击后显示提示或对白 |
| 拾取物 | `pickup_object.tscn` | 点击后进入背包并隐藏 |
| 视角切换 | `view_exit_object.tscn` | 点击切换到另一个房间视角 |
| 剧情触发 | `story_trigger_object.tscn` | 点击触发 NarrRail event |
| 谜题入口 | `puzzle_entry_object.tscn` | 点击打开谜题/小游戏 |
| 锁定物 | `locked_object.tscn` | 需要指定背包物品才能成功交互 |

Inspector 里如果看不到某个 action 字段，通常说明当前对象类型不应该配置它。需要完全自定义动作链时，改用通用 `interactive_object.tscn`。

## 四、快捷字段

常见交互可以不手写 action，直接填快捷字段：

- `pickup_item_id`：点击后拾取的物品 ID。
- `story_event_on_click`：点击后触发的 NarrRail 故事名或 `StoryBridge` 映射 key。
- `target_view_id`：点击后切换到的视角 ID。
- `target_scene_path`：点击后切换到的场景路径。
- `puzzle_id`：点击后打开的谜题 ID。
- `toast_message`：点击后的提示文本。
- `required_item_id`：成功交互需要的背包物品 ID。
- `missing_item_toast_message`：没有选中需求物品时显示的提示文本。
- `consume_required_item`：成功后是否消耗需求物品。
- `set_flag_on_success`：成功后设置为 `true` 的 flag。
- `state_on_success`：成功后写入交互物存档状态的 `state`。

如果这些字段不够用，再配置 `default_action_resources` 或 `item_interaction_resources`。

## 五、Action Resource 配置

推荐使用 Resource action 代替手写 Dictionary。当前可用类型：

```gdscript
PickupItemAction
RemoveItemAction
SetFlagAction
ChangeViewAction
EmitStoryEventAction
ShowDialogueAction
ShowToastAction
OpenPuzzleAction
SetObjectStateAction
```

在 Inspector 里配置：

- `default_action_resources`：空手点击或默认点击执行的动作链。
- `item_interaction_resources`：选中特定背包物品后执行的动作链。

旧字段 `default_actions` 和 `item_interactions` 仍然兼容，但后续新增内容优先使用 Resource action。

## 六、拾取物品

最简单做法：

- 使用 `pickup_object.tscn`。
- 填 `pickup_item_id = prototype_key`。
- 填 `toast_message = 获得钥匙`。

效果：

- 物品加入右侧背包。
- 场景里的物品隐藏。
- 自动存档。

## 七、普通查看物

使用 `inspect_object.tscn`，填 `toast_message`。

## 八、触发 NarrRail 对白

使用 `story_trigger_object.tscn`，填 `story_event_on_click`。

如果故事是从 NarrRail 仓库同步到 Godot 的 `.tres`，直接填写文件名去掉后缀：

```text
res://narrrail_stories/HijackSoul_Stories/Stories/train_draft.tres
```

对应填写：

```text
story_event_on_click = train_draft
```

如果需要使用别名，也可以填 `StoryBridge.story_event_map` 里的 key，由程序映射到具体故事资源路径。

## 九、需要背包物品的交互

例如一个锁盒：

- 空手点击：提示锁住。
- 选中钥匙点击：消耗钥匙并打开。

快捷字段做法：

- 使用 `locked_object.tscn`。
- `required_item_id = prototype_key`
- `missing_item_toast_message = 盒子锁住了。`
- `consume_required_item = true`
- `set_flag_on_success = box_opened`
- `state_on_success = open`
- `toast_message = 盒子打开了。`

如果失败提示和成功提示需要不同文本，使用 `default_action_resources` 配置失败提示，再用 `item_interaction_resources` 配置成功动作链。

## 十、命名规则

`object_id` 建议格式：

```text
章节.房间.物品
```

例：

```text
chapter_01.room_01.key
chapter_01.room_01.locked_box
chapter_01.room_01.wall_note
```

注意：

- `object_id` 不能重复。
- 已经上线或已经用于存档的 `object_id` 不要随便改。
- 可拾取物品的 `item_id` 也要稳定。

## 十一、检查清单

新增交互物后，请确认：

- [ ] 放在当前场景的 `ObjectLayer` 下。
- [ ] 使用了合适的细分预制体。
- [ ] 填了唯一的 `object_id`。
- [ ] 填了 `display_name` 和 `hover_text`。
- [ ] 设置了 `hotspot_size`，鼠标容易点中。
- [ ] 配置了快捷字段或 Resource action。
- [ ] 运行后 hover 有提示。
- [ ] 点击后反馈符合预期。
