# 故事事件协议

本文档记录已经接入 HijackSoul 的 NarrRail 故事事件、参数规格和游戏内作用。

## 事件格式

所有新 NarrRail 故事事件必须使用结构化 `eventType + params`：

```yaml
- nodeId: N_Event
  nodeType: EmitEvent
  eventType: ui.show_toast
  params:
    message: Story event triggered
```

不要为新故事继续使用旧 `eventId` 字段。`eventType` 表示事件名称，`params` 是该事件的参数对象。

运行时流向：

```text
NarrRail EmitEvent
-> StoryBridge
-> StoryEventRouter
-> ActionRunner / EventBus / 具体 UI 或模块
```

未登记的 `eventType` 会被记录为未处理，但不会打断剧情。已登记事件缺少必需参数时会报错，并触发 `story_event_failed`。

## 已实现事件总表

| 事件名称 | 配置表短名 | 必填参数 | 可选参数 | 当前状态 | 游戏内作用 |
| --- | --- | --- | --- | --- | --- |
| `dialogue.change_face` | `change_face` | `role`, `face` | 无 | 已注册 | 发出角色表情切换请求。具体显示效果需要角色表现层订阅。 |
| `dialogue.change_posture` | `change_posture` | `role`, `posture` | 无 | 已注册 | 发出角色姿势/动作状态切换请求。具体显示效果需要角色表现层订阅。 |
| `dialogue.set_npc_bubble_position` | `set_npc_bubble_position` | `x`, `y` 或 `position.x`, `position.y` | `speakerId`, `side` | 已接入 UI | 设置 NPC 对话气泡位置和尾巴方向。 |
| `dialogue.set_replay_line` | `set_replay_line` | `line_id`, `text` | `speaker` | 已注册 | 登记二周目或回放替换台词。具体替换显示需要叙事/UI 层订阅。 |
| `object.play_animation` | `play_animation` | `object`, `animation` | 无 | 已注册 | 发出对象播放动画请求。具体播放需要对象/动画系统订阅。 |
| `object.set_state` | `set_object_state` | `object`，以及至少一个状态字段 | `visible`, `enabled`, `state`, `picked`, `extra` | 已接入 GameState | 设置场景交互物的通用状态，并触发 story event autosave 链路。 |
| `object.set_visible` | `set_object_visible` | `object`, `visible` | 无 | 已接入 GameState | 设置场景交互物显示或隐藏，并触发 story event autosave 链路。 |
| `object.switch_material` | `switch_material` | `resource`, `material` | 无 | 已注册 | 发出素材/材质切换请求。具体替换需要表现系统订阅。 |
| `matter.inspect` | `inspect_matter` | `object`, `matter` | 无 | 已注册 | 发出检视 matter 请求。具体检视 UI 或数据系统需要后续订阅。 |
| `audio.play_sound` | `play_sound` | `sound` | `object` | 已注册 | 发出播放音效请求。具体播放需要音频系统订阅。 |
| `scene.jump` | `jump_scene` | `scene` | `scene_path` | 已接入 SceneFlow | 请求切换到指定 scene/view。目标必须已在 `SceneFlowManager` 注册，或传入 `scene_path`。 |
| `state.set_flag` | 无 | `flag_id` | `value` | 已接入 GameState | 设置 `GameState` flag，并触发 story event autosave 链路。 |
| `ui.show_toast` | 无 | `message` | 无 | 已接入 UI 请求 | 显示一条 toast 提示文本，并触发 story event autosave 链路。 |
| `environment.set_state` | `set_environment_state` | `environment`, `state` | `transition_seconds`, `note` | 已注册 | 发出环境状态切换请求，例如窗外景色、灯光状态。具体表现需要场景系统订阅。 |
| `journal.unlock_entry` | `unlock_journal_entry` | `entry_id`, `text` | `source` | 已接入 GameState | 解锁卧室笔记/物证文本，写入 `journal_entry.<entry_id>` flag，并触发 autosave。 |
| `timeline.wait` | `wait` | `seconds` | 无 | 已接入 StoryEventRouter | 暂停故事推进指定秒数。 |

配置表短名是兼容写法，`StoryEventRouter` 会接受短名和正式事件名。正式 `.nrstory` 建议优先使用带命名空间的事件名。

`object` 参数应填写场景物体手动配置的稳定 `object_id`，不是 Godot 节点名。比如节点名可能是 `WallNote`，但事件里应填 `prototype.room_left.wall_note`。`object_id` 会进入存档和状态恢复流程，已经上线或用于存档后不要随意改名。

## `dialogue.change_face`

切换角色表情或立绘状态。

当前实现状态：
- `StoryEventRouter` 已校验参数并发出 `EventBus.dialogue_face_change_requested(role_id, face_id, payload)`。
- 具体角色立绘/表情显示模块还需要订阅该信号后，才会真正改变画面。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `role` | String | 是 | 无 | 角色 ID，例如 `girl`。兼容别名：`role_id`。 |
| `face` | String | 是 | 无 | 表情或立绘状态 ID，例如 `expression_girl_expressionless`。兼容别名：`face_id`。 |

示例：

```yaml
- nodeId: N_GirlExpressionless
  nodeType: EmitEvent
  eventType: dialogue.change_face
  params:
    role: girl
    face: expression_girl_expressionless
```

游戏内作用：
- 广播“把 `girl` 的表情切到 `expression_girl_expressionless`”的请求。
- 不修改 `GameState`。
- 不直接触发 autosave。

兼容配置表写法：

```text
{ change_face : role = girl , face = expression_girl_expressionless }
```

## `dialogue.set_npc_bubble_position`

设置 NPC 对话气泡在画面上的位置。

当前实现状态：
- `StoryEventRouter` 已校验坐标参数。
- 事件会发出 `EventBus.dialogue_npc_bubble_position_requested(payload)`。
- `StoryBridge` 会继续转发到现有 `dialogue_npc_bubble_position_requested` 信号，当前 `dialogue_overlay` 已接入该信号。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `x` | Number | 是 | 无 | 气泡锚点横坐标。可用 `position.x` 替代。 |
| `y` | Number | 是 | 无 | 气泡锚点纵坐标。可用 `position.y` 替代。 |
| `position` | Object | 否 | 无 | 坐标对象，格式为 `{ x: Number, y: Number }`。 |
| `speakerId` | String | 否 | 空字符串 | 指定说话人。兼容别名：`speaker_id`。为空时更新默认 NPC 气泡位置。 |
| `side` | String | 否 | `right` | 气泡尾巴方向。当前 UI 侧支持 `left`、`right`、`up`、`down`。 |

示例：

```yaml
- nodeId: N_MoveGirlBubble
  nodeType: EmitEvent
  eventType: dialogue.set_npc_bubble_position
  params:
    speakerId: girl
    x: 760
    y: 260
    side: right
```

等价坐标写法：

```yaml
- nodeId: N_MoveGirlBubble
  nodeType: EmitEvent
  eventType: dialogue.set_npc_bubble_position
  params:
    speakerId: girl
    position:
      x: 760
      y: 260
    side: right
```

游戏内作用：
- 如果 `speakerId` 为空，更新默认 NPC 气泡位置。
- 如果 `speakerId` 非空，只更新该说话人的气泡位置。
- 当前 NPC 气泡正在显示且说话人匹配时，会立即重新定位。
- 不修改 `GameState`。
- 不直接触发 autosave。

## `object.set_state`

设置场景交互物的通用状态，适合“出现但不可交互”“提示可交互”“已被拾取/消失”等不是单纯显隐的状态。

当前实现状态：
- `StoryEventRouter` 会把事件转换成 `ActionRunner` 的 `set_object_state` action。
- 继承/使用 `InteractiveObject` 的场景物体会应用 `visible`、`enabled`、`state`、`picked` 字段。
- action chain 完成后会通过 `save_reason = story_event` 请求 autosave。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `object` | String | 是 | 无 | 目标场景物体稳定 `object_id`。兼容别名：`object_id`。 |
| `visible` | Bool | 否 | 无 | 是否显示。 |
| `enabled` | Bool | 否 | 无 | 是否可交互。 |
| `state` | String | 否 | 无 | 业务状态字符串，例如 `hint_enabled`。 |
| `picked` | Bool | 否 | 无 | 为 `true` 时当前 `InteractiveObject` 会隐藏并禁用。 |
| `extra` | Object | 否 | 无 | 额外状态字段，会一起写入 `GameState.object_states`。 |

示例：

```yaml
- nodeId: N_ShowBook
  nodeType: EmitEvent
  eventType: object.set_state
  params:
    object: train.book_biancheng
    visible: true
    enabled: false
```

兼容配置表写法：

```text
{ set_object_state: object = train.book_biancheng , visible = true , enabled = false }
```

## `object.set_visible`

设置场景交互物显示或隐藏。

当前实现状态：
- `StoryEventRouter` 会把事件转换成 `ActionRunner` 的 `set_object_state` action。
- `ActionRunner` 调用 `GameState.set_object_state(object_id, {"visible": visible})`。
- 继承/使用 `InteractiveObject` 的场景物体会监听 `GameState.object_state_changed`，并立即应用 `visible`。
- action chain 完成后会通过 `save_reason = story_event` 请求 autosave。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `object` | String | 是 | 无 | 目标场景物体的稳定 `object_id`，例如 `prototype.room_left.wall_note`。兼容别名：`object_id`。 |
| `visible` | Bool | 是 | 无 | `true` 表示显示，`false` 表示隐藏。 |

隐藏示例：

```yaml
- nodeId: N_HideWallNote
  nodeType: EmitEvent
  eventType: object.set_visible
  params:
    object: prototype.room_left.wall_note
    visible: false
```

显示示例：

```yaml
- nodeId: N_ShowWallNote
  nodeType: EmitEvent
  eventType: object.set_visible
  params:
    object: prototype.room_left.wall_note
    visible: true
```

兼容配置表写法：

```text
{ set_object_visible: object = prototype.room_left.wall_note , visible = false }
```

游戏内作用：
- 把 `prototype.room_left.wall_note` 的对象状态写成 `visible = false` 或 `visible = true`。
- 当前场景里同 `object_id` 的 `InteractiveObject` 会立即显示或隐藏。
- 状态会随 `GameState` 存档恢复。
- 请求一次 `story_event` 原因的 autosave。

## `scene.jump`

请求跳转到指定场景或视角。

当前实现状态：
- `StoryEventRouter` 会把事件转换成 `ActionRunner` 的 `change_view` action。
- `ActionRunner` 调用 `SceneFlowManager.change_view(view_id, scene_path)`。
- 如果 `scene_path` 为空，目标 `scene` 必须已经在 `SceneFlowManager.view_paths` 中注册。
- action chain 完成后会通过 `save_reason = story_event` 请求 autosave。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `scene` | String | 是 | 无 | 目标场景或视角 ID，例如 `study`。兼容别名：`scene_id`、`view_id`。 |
| `scene_path` | String | 否 | 空字符串 | 目标 Godot 场景路径，例如 `res://levels/study/study.tscn`。传入时会先注册到 `SceneFlowManager`。 |

示例：

```yaml
- nodeId: N_JumpStudy
  nodeType: EmitEvent
  eventType: scene.jump
  params:
    scene: study
```

带场景路径示例：

```yaml
- nodeId: N_JumpStudy
  nodeType: EmitEvent
  eventType: scene.jump
  params:
    scene: study
    scene_path: res://levels/study/study.tscn
```

兼容配置表写法：

```text
{ jump_scene: scene = study }
```

游戏内作用：
- 请求切换到 `study` 视角或场景。
- 成功后 `SceneFlowManager` 会更新当前 view。
- 请求一次 `story_event` 原因的 autosave。

请求环境表现切换，例如车窗外景色、车厢灯光、隧道状态或月台进入。

当前实现状态：
- `StoryEventRouter` 已校验参数并发出 `EventBus.environment_state_change_requested(environment_id, state_id, payload)`。
- 具体视觉表现需要场景或表现系统订阅该信号。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `environment` | String | 是 | 无 | 环境通道 ID，例如 `train.window_view` 或 `train.lighting`。兼容别名：`environment_id`。 |
| `state` | String | 是 | 无 | 目标状态 ID，例如 `field`、`sea`、`dark_tunnel`。兼容别名：`state_id`。 |
| `transition_seconds` | Number | 否 | 无 | 表现层可选使用的过渡秒数。 |
| `note` | String | 否 | 无 | 给表现层或策划调试用的备注。 |

兼容配置表写法：

```text
{ set_environment_state: environment = train.window_view , state = sea , transition_seconds = 20 }
```

## `journal.unlock_entry`

解锁卧室笔记、物证、回忆文本等跨场景叙事条目。

当前实现状态：
- `StoryEventRouter` 已校验参数并发出 `EventBus.journal_entry_unlocked_requested(entry_id, text, payload)`。
- 同时写入 `GameState` flag：`journal_entry.<entry_id> = { unlocked = true, text = ..., source = ... }`。
- action chain 完成后会通过 `save_reason = story_event` 请求 autosave。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `entry_id` | String | 是 | 无 | 条目稳定 ID。兼容别名：`entry`。 |
| `text` | String | 是 | 无 | 解锁文本。 |
| `source` | String | 否 | 空字符串 | 来源，例如 `bedroom_notebook`。 |

兼容配置表写法：

```text
{ unlock_journal_entry: entry_id = sea_end_note , source = bedroom_notebook , text = 想要去看海的尽头 }
```

## `timeline.wait`

暂停故事推进指定秒数，用于表演、噪音、灯光变化后的留白。

当前实现状态：
- `StoryEventRouter` 会等待 `params.seconds` 秒后返回。

参数规格：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `seconds` | Number | 是 | 无 | 等待秒数，必须大于 0。兼容别名：`duration`。 |

兼容配置表写法：

```text
{ wait: seconds = 5 }
```

## 新增事件规则

1. 先在本文档登记事件名称、参数规格和游戏内作用。
2. 在 `hijacksoul/autoload/story_event_router.gd` 添加 handler 和参数校验。
3. 表现层事件优先走 `EventBus`，由 UI 或表现模块订阅。
4. 状态类、物品类、场景类事件优先转换成 `ActionRunner` action。
5. 项目专属事件语义不要写进 `addons/narrrail` 插件。
