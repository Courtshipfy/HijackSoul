# 项目架构文档

**Godot 数据驱动游戏框架**

> 当前阶段：主题未公布 · 基础搭建中

---

## 核心原则

策划改 Excel → 工具转 JSON → 游戏读取 → 模块处理 → UI 显示

---

## 一、目录结构

```
📁 项目仓库/
│
├── 📁 _Assets/              ← 源文件区，游戏不读取
│   ├── 📁 Art/              🎨 美术（PSD/源文件 → 导出 PNG）
│   ├── 📁 Design/           📝 策划（Excel 表 + 文档）
│   ├── 📁 Audio/            🔊 音频（源文件）
│   └── 📁 Pipeline/         ⚙️ 规范（命名/导出/数据规则）
│
├── 📁 _Tools/               💻 程序（Excel 转 JSON 工具，策划双击运行）
│
└── 📁 GodotProject/         💻 程序（Godot 工程）
    ├── 📁 assets/           🎨 美术最终资源（游戏读取）
    ├── 📁 data/             📝 策划 JSON 数据（游戏读取，只读）
    ├── 📁 autoload/         💻 全局服务（事件/场景/音效/存档/数据/状态）
    ├── 📁 modules/          💻 玩法模块（玩家/敌人/背包/任务/对话/战斗/交互）
    ├── 📁 ui/               💻 界面（菜单/HUD/背包/对话）
    ├── 📁 levels/           💻 关卡（只摆放，不写逻辑）
    ├── 📁 glue/             💻 胶水（连接模块，不写玩法）
    └── 📁 tools/            💻 Godot 编辑器插件
```

---

## 二、谁负责什么

| 人员 | 负责 | 不碰 |
|------|------|------|
| 🎨 美术 | `_Assets/Art/`、`GodotProject/assets/` | `data/`、代码 |
| 📝 策划 | `_Assets/Design/`、Excel 表 + 文档 | `GodotProject/` 任何代码或场景 |
| 🔊 音频 | `_Assets/Audio/`、`assets/audio/` | 代码、逻辑 |
| 💻 程序 | 所有代码和场景 | `data/` 手改、`_Assets/` 源文件 |

---

## 三、三条铁律

### 1. 策划只改 Excel，不手改 JSON

改 Excel → 运行转换器 → 生成 JSON → 游戏自动读取

### 2. 模块之间不直接引用，只发信号

player 不直接调用 `inventory.add()`，而是发 EventBus 信号

### 3. glue 只连接，不写玩法

胶水层转发信号，不做伤害计算、不做 AI 决策

---

## 四、数据流

```
策划 Excel → 转换器 → JSON → data/ → 游戏读取 → modules 处理 → UI 显示

美术源文件 → 导出 → assets/ → 游戏读取 → 显示
```

## 五、当前阶段做什么（主题未公布）

### ✅ 现在做

- 搭 `autoload/` 全局服务框架
- 搭 player 空节点树 + UI 菜单
- 写 Excel 转 JSON 工具
- 定命名规范和数据规范

### ⛔ 等主题公布再做

- 写具体玩法（AI、伤害、任务条件）
- 设计关卡内容
- 填正式 Excel 表
- 做正式美术资源

---

## 六、主题公布后怎么接入

1. 确定玩法循环（如：潜行→收集→逃脱）
2. 填充 EventBus 事件（`player_died`、`item_collected`）
3. 填充 `modules/` 逻辑（移动、背包、对话）
4. 填充 `glue/` 连接（信号转发）
5. 填充 `levels/` 摆放（地图、敌人、道具）
6. 策划填 Excel，转 JSON 进游戏
