# 场景视角切换配置说明

本文档给策划使用，说明如何配置锈湖式的房间视角切换。

## 一、基本概念

一个“房间”可以有多个“视角”。

例如：

```text
room_01_front    正面视角
room_01_left     左侧视角
room_01_drawer   抽屉近景
```

当前做法是：**每个视角对应一个 Godot 场景文件**。

## 二、视角场景放在哪里

prototype 示例：

```text
res://levels/prototype/prototype_room_front.tscn
res://levels/prototype/prototype_room_left.tscn
```

正式章节建议：

```text
res://levels/chapter_01/room_01_front.tscn
res://levels/chapter_01/room_01_left.tscn
res://levels/chapter_01/room_01_drawer.tscn
```

## 三、视角场景必须配置什么

选中视角场景根节点 `RoomView`，配置：

- `view_id`：当前视角 ID。
- `scene_path`：当前场景路径。

例：

```text
view_id = chapter_01.room_01.front
scene_path = res://levels/chapter_01/room_01_front.tscn
```

## 四、如何添加切换按钮

在当前视角的 `ObjectLayer` 下拖入：

```text
res://modules/interaction/interactive_object.tscn
```

把它摆到画面边缘、门、抽屉、柜子等可点击位置。

常用字段：

- `object_id`：唯一 ID。例：`chapter_01.room_01.exit_left`
- `display_name`：目标名称。例：`左侧`
- `hover_text`：悬停提示。例：`看向左侧`
- `hotspot_size`：点击范围。

## 五、切换到另一个视角

在切换按钮的 `default_actions` 里写：

```gdscript
[
  {
    "type": "change_view",
    "view_id": "chapter_01.room_01.left",
    "scene_path": "res://levels/chapter_01/room_01_left.tscn"
  }
]
```

效果：

- 点击该热点后切到指定视角。
- 当前视角会写入存档状态。

## 六、返回原视角

在目标视角里也要放一个返回热点。

例：左侧视角返回正面视角：

```gdscript
[
  {
    "type": "change_view",
    "view_id": "chapter_01.room_01.front",
    "scene_path": "res://levels/chapter_01/room_01_front.tscn"
  }
]
```

## 七、近景视角

近景也按普通视角处理。

例：点击抽屉进入抽屉近景：

```gdscript
[
  {
    "type": "change_view",
    "view_id": "chapter_01.room_01.drawer",
    "scene_path": "res://levels/chapter_01/room_01_drawer.tscn"
  }
]
```

抽屉近景里再放一个返回按钮：

```gdscript
[
  {
    "type": "change_view",
    "view_id": "chapter_01.room_01.front",
    "scene_path": "res://levels/chapter_01/room_01_front.tscn"
  }
]
```

## 八、命名建议

`view_id` 建议格式：

```text
章节.房间.视角
```

例：

```text
chapter_01.room_01.front
chapter_01.room_01.left
chapter_01.room_01.drawer
```

切换热点的 `object_id` 建议格式：

```text
chapter_01.room_01.exit_目标视角
```

例：

```text
chapter_01.room_01.exit_left
chapter_01.room_01.exit_drawer
```

## 九、检查清单

新增视角切换后，请确认：

- [ ] 目标视角场景文件已经存在。
- [ ] 目标视角根节点填了正确的 `view_id`。
- [ ] 当前热点的 `scene_path` 路径正确。
- [ ] 当前热点的 `view_id` 和目标视角一致。
- [ ] 热点 `hotspot_size` 足够大，容易点中。
- [ ] 目标视角里有返回热点。
- [ ] 运行后可以来回切换。
